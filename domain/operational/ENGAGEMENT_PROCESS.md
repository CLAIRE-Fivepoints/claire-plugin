# Five Points – Engagement Process

## Phase 1: Onboarding (current)

1. Receive reference materials from Five Points (see CLIENT_GUIDE.md checklist)
2. Short working session with Steve Franklin to walk through codebase (30–45 min)
3. Validate that reference code covers full stack vertical

## Phase 2: PoC Execution

1. Calibrate C.L.A.I.R.E. to Five Points' patterns using reference code
2. Generate code for selected PBIs
3. Submit for review by Steve Franklin

## Phase 3: Evaluation

1. Five Points team evaluates output quality vs. current code-gen process
2. Calibration loop if needed
3. Decision on commercial arrangement

## Key Success Criteria

- Generated code matches Five Points' architectural patterns
- Code passes their review without major rework
- Delivery speed comparable or better than their current few-hours-per-PBI benchmark

## Notes

- Do not start Option 2 before Option 1 is validated
- All technical questions go through Steve Franklin
- Commercial decisions go through Brian Cliburn
